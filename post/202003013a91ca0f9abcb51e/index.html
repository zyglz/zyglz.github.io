Gridea+Github快速搭建个人博客教程

置顶 | 发布于 2020-03-01 | 分类于 Github 、 个人博客 、 Gridea | 4分钟 | 997字数

👏想知道如何使用Gridea吗？如何使用Gridea在Github上搭建个人博客吗？

✍️Gridea是一个静态博客写作客户端。你可以用它来记录你的生活、心情、知识、笔记、创意。

🍀Github👇

📝 Github官网：https://github.com

⛳️ 如果有账号，直接登录，没有账号注册一个

⛳️ 每个github账号有1个免费的github pages，默认域名即为：用户名.github.io,如果免费的已被占用（例如zyglz），得自行注册新的github账号，或者自行买域名，在仓库里的github pages里设置好，做好域名映射即可。

🌎仓库

  1. 点击create a new repositories
  2. 输入Repository name：Cyberlisa.github.io。固定格式：.github.io
  3. Description可填可不填。
  4. 勾选Public
  5. 勾选Intialize this repository with a README(可不勾选)
  6. 点击Create repository
  7. 自动跳转到仓库页面，点击Insights后面的Settings
  8. 拉到下面GitHub Pages，看看有没有显示：Your site is published at https://zyglz.github.io/
  9. 没有的话稍微等一下再试
  10. 有的话点击链接跳转，看看能否打开

🖥Token

  1. 在github界面点击头像，点击Settings——>Personal settings——>Developer settings
  2. 找到最后的Personal access tokens——>Generate new token——>填写Note——>勾选repo——>Generate token
  3. 建议新建一个txt文档，把token保存起来。因为token值仅且只显示一次，关闭窗口后再无法看到。如果不保存，那先不要关闭这个页面，待会儿需要用到token。

🌁 Gridea配置

💻 下载安装

🚲在https://gridea.dev下载符合自己系统的客户端，安装。
🚲安装完之后打开，默认有两篇文章：about 和hello-gridea；如图：

🌱配置Gridea

✈️打开后，点击【远程】：

名称 填写说明 举例
选择Github Pages
域名 Github Pages 对应的域名 https://zyglz.github.io/
仓库 github上仓库名 zyglz.github.io
分支 Github Pages 对应的分支 master
仓库用户名 仓库所属用户的用户名 zyglz
邮箱 Git 推送时的邮箱 25851616@qq.com
Token Token 上面步骤里的token
CNAME 域名 可不填
  1. 填完配置后，点击【保存】
  2. 无误后，点击【检测远程连接】
  3. 点击【预览】，会弹出浏览器，本地预览效果。
  4. 可在【主题】里选择其他内置主题，不要忘记拉到下面选择【保存】
  5. 选择了其他主题后，点击【预览】
  6. 如果没问题，点击【同步】
  7. 稍等3分钟左右，点击同步按钮下面那个像地球的网格图标，会在浏览器自动弹出博客网页。看看有没有什么问题。
  8. 在客户端里进行任何操作，不要忘记点击【保存】。保存了之后先预览，没问题再同步。
  9. 有问题点击https://github.com/getgridea/gridea里的【Issues】，在【Open】和【Closed】里看看有没有和你问题相似的issue。

🌉更换主题

🗼在https://gridea.dev/themes/，寻找你喜欢主题，然后点击【下载文件】，下载后解压。
🗼点击客户端里的【配置】，找到【 站点源文件路径 】（我的为：D:/Users/Administrator/Documents/Gridea），打开路径。退出Gridea客户端。
🗼选择【themes】文件夹，把刚刚解压的主题文件夹复制过来，重新打开Gridea客户端，打开【主题】，选择主题，滚动下鼠标，选中刚刚复制进去的主题名称，保存。
🗼点击预览，即可看到新的主题页面。

💬 开启Gitalk评论系统

🏷️在https://github.com/settings/applications/new注册应用。

Application name Gridea_gitalk（自定义）
Homepage URL https://zyglz.github.io/
Application description 可不填
Authorization callback URL https://zyglz.github.io/

🏷️填写后点击【Register application】，会得到【Client ID】和【Client Secret】。

🏷️打开Gridea客户端->远程->评论配置->平台（选中Gitalk）->按照下面提示填写->

名称 内容
Client ID
Client Secret
仓库 zyglz.github.io
Owner zyglz

🏷️填完后点击【保存】->【同步】。

📋稍等一会儿后，在浏览器打开域名，会看到文章底下显示“未找到相关的issues进行评论xxx”，点击【使用GitHub登陆】，进行授权。
